# Security Policy

## Supported Versions

This repository contains portfolio projects. Only the `main` branch of each project receives security attention.

## Reporting a Vulnerability (Responsible Disclosure)

**Please do not open public issues for security vulnerabilities.**

1. **Report privately** via GitHub's [Private Vulnerability Reporting](https://github.com/nadidoug/nadi-portfolio/security/advisories/new) or email **nadi@nadidoug.com** with subject line `SECURITY:`.
2. Include: affected file/project, reproduction steps, impact assessment, and any suggested fix.
3. **Response targets:** acknowledgment within 72 hours; triage within 7 days; fix or mitigation timeline communicated within 14 days.

## Safe Harbor

If you make a good-faith effort to comply with this policy — no data exfiltration, no service disruption, no access beyond what's needed to demonstrate the issue — I will not pursue legal action regarding your research and will credit you (if desired) in the fix notes.

## Scope

In scope: code in this repository and the nadidoug.com website.
Out of scope: third-party services, social engineering, physical attacks, denial-of-service testing.
